# 音頻穩定性改進說明

## 問題診斷

用戶報告聲音變形問題，可能的原因包括：

1. **重採樣失真**：從 44.1kHz/48kHz 降採樣到 16kHz 時，抗混疊濾波不當
2. **PCM 轉換失真**：Float32 轉 PCM16 時的量化誤差
3. **音量不穩定**：沒有音量標準化，導致音頻忽大忽小
4. **直流偏移**：音頻中的直流分量導致失真
5. **過度過濾**：靜音閾值過高，切掉了有效音頻

## 改進方案

### 1. 改進重採樣算法 ✅

**問題**：
- 簡單的線性插值沒有足夠的抗混疊保護
- 高通濾波實現錯誤（實際上是低通濾波）

**解決方案**：
- 使用改進的線性插值 + 移動平均低通濾波
- 正確實現一階高通濾波器去除直流偏移
- 計算抗混疊濾波器的截止頻率（Nyquist 頻率的一半）

**代碼位置**：`useElevenLabsConvAI.ts` 第 480-540 行

### 2. 優化 PCM 轉換 ✅

**問題**：
- 靜音閾值 0.01 過高，會切掉有效音頻
- 沒有音量標準化，導致音頻忽大忽小
- 量化誤差累積

**解決方案**：
- 添加動態範圍控制：根據 RMS 自動調整增益
- 降低靜音閾值到 0.0005（只過濾真正的背景噪音）
- 使用更精確的轉換算法（Math.fround + Math.round）

**代碼位置**：`useElevenLabsConvAI.ts` 第 70-86 行

### 3. 音量標準化和動態範圍控制 ✅

**新增功能**：
- 計算音頻的 RMS（均方根）和峰值
- 自動增益控制：
  - 如果 RMS < 0.05：輕微放大（最多 2 倍）
  - 如果 RMS > 0.5：輕微壓縮（避免削波）
  - 目標 RMS：0.1-0.3（最佳範圍）

**好處**：
- 保持音頻音量穩定
- 避免削波失真
- 提高小聲音的清晰度

### 4. 改進音頻處理鏈 ✅

**改進**：
- 移除重複的靜音閾值過濾（在重採樣後）
- 將靜音過濾統一在 PCM 轉換階段處理
- 減少處理步驟，降低累積誤差

## 技術細節

### 重採樣算法

```typescript
// 1. 計算抗混疊濾波器參數
const ratio = SAMPLE_RATE / ctxSampleRate;
const nyquistFreq = SAMPLE_RATE / 2;
const cutoffFreq = Math.min(nyquistFreq * 0.95, ctxSampleRate / 2 * 0.95);

// 2. 線性插值
const value = inputData[srcIndexFloor] * (1 - t) + inputData[srcIndexCeil] * t;

// 3. 移動平均低通濾波（減少混疊）
value = value * 0.7 + (windowAverage) * 0.3;

// 4. 高通濾波（去除直流偏移）
const lowpass = prevLowpass * alpha + value * (1 - alpha);
output = value - lowpass;
```

### PCM 轉換算法

```typescript
// 1. 計算 RMS 和峰值
const rms = Math.sqrt(sumSquares / length);
const maxAbs = Math.max(...absValues);

// 2. 動態增益控制
let gain = 1.0;
if (rms < 0.05) gain = Math.min(2.0, 0.1 / rms);
else if (rms > 0.5) gain = Math.max(0.5, 0.5 / rms);

// 3. 應用增益並轉換
const scaled = (value * gain) * 32768.0;
pcm16[i] = Math.round(scaled);
```

## 測試建議

1. **音質測試**：
   - 測試不同音量級別的語音
   - 測試快速說話和慢速說話
   - 測試背景噪音環境

2. **穩定性測試**：
   - 長時間對話（10+ 分鐘）
   - 檢查音頻是否持續穩定
   - 檢查是否有突然的音量變化

3. **性能測試**：
   - 檢查 CPU 使用率（重採樣可能增加計算負擔）
   - 檢查音頻延遲

## 回滾方案

如果改進後出現問題，可以通過以下方式回滾：

1. 恢復舊的重採樣算法（簡單線性插值）
2. 恢復舊的 PCM 轉換（無動態增益控制）
3. 恢復靜音閾值過濾（在重採樣後）

**注意**：建議先測試新版本，確認改進效果後再決定是否回滾。
